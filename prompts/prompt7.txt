## Clasificación de errores

| Categoría | Síntomas | Dónde buscar |
|-----------|----------|---------------|
| **Importación** | `ModuleNotFoundError`, `ImportError` | Rutas, `sys.path`, `conftest.py` |
| **Base de datos** | `sqlite3.OperationalError`, tabla no encontrada | `DB_PATH`, `crear_tabla()`, migraciones |
| **Interfaz gráfica** | Ventana no abre, widgets no se ven | Layouts, `show()`, `setCentralWidget` |
| **Lógica/Validación** | `ValueError`, datos incorrectos | Controladores, modelos |
| **Pruebas** | `AssertionError`, pruebas que no se ejecutan | Nombres `test_*.py`, `pytest.ini` |
| **Entorno** | `pip install` falla, versión incorrecta | `requirements.txt`, Python, venv |

## ERROR 1: MODULENOTFOUNDERROR

### Síntomas
ModuleNotFoundError: No module named 'src'
ModuleNotFoundError: No module named 'PySide6'

text

### Causas comunes
1. No se ejecuta desde la raíz del proyecto
2. El entorno virtual no está activado
3. Falta instalar dependencias
4. `conftest.py` no tiene las rutas correctas

### Soluciones

```bash
# 1. Verificar directorio actual
cd D:\02. Proyectos\Proyectos Desktop\SISVENIN

# 2. Activar entorno virtual

pip install -r requirements.txt

ERROR 4: PYSIDE6 VENTANA NO ABRE
Síntomas
La terminal muestra "Process finished" pero no hay ventana

La ventana aparece y se cierra inmediatamente

Causas comunes
Falta app.exec() o sys.exit(app.exec())

show() antes de exec()

Error en main.py

Soluciones
python
# src/main.py debe tener esta estructura CORRECTA:

import sys
from PySide6.QtWidgets import QApplication
from src.app.app import App

def main():
    app = QApplication(sys.argv)
    ventana = App()
    ventana.show()
    sys.exit(app.exec())  # ← fundamental!

if __name__ == "__main__":
    main()
python
# Si la ventana se cierra sola, verificar que App hereda de QMainWindow

# src/app/app.py

    ...

ERROR 9: EL BOTÓN NO HACE NADA
Síntomas
Se hace clic en un botón pero no ocurre nada.

Causas comunes
La señal clicked no está conectada

El método conectado no existe

Error en el método (no visible por no haber try/except)

Soluciones
python
# 1. Verificar conexión
self.btn_nuevo.clicked.connect(self.nuevo)  # ← debe existir

# 2. Verificar que el método existe
def nuevo(self):  # ← debe existir con ese nombre
    print("DEBUG: Botón clickeado")  # ← debug temporal
    QMessageBox.information(self, "Info", "Funcionalidad en desarrollo")

# 3. Agregar try/except para ver errores
def nuevo(self):
    try:
        print("DEBUG: Botón clickeado")
        # código aquí
    except Exception as e:
        QMessageBox.critical(self, "Error", str(e))
